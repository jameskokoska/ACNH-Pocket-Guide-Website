App Data

Importing/Exporting Data to Local Storage Permissions

The (Expo React Native) SDK and the implementation of exporting data require the user to specify a local folder on their device. We use the latest Android library, meaning only the folder you specify the app will be able to write the file to. Importing data requires the user to specify a local file on their device. The app will only read and only has access to that selected file.

Importing/exporting is optional. The app will not request any additional permissions beyond what is necessary for the features you choose to use.

Stored Data (On Device)

Your in-app data is stored locally on your device using the React Native Async Storage library:
https://github.com/react-native-async-storage/async-storage

Local data can be cleared by going to:
Settings > Reset Data > Reset
(this cannot be undone)
Or by clearing application data and cache in your device settings.

Cloud Backup (Optional)

If you enable and use Cloud Backup, the app will collect and transmit data to provide that service. Cloud Backup is maintained using Google Firebase services.

What we collect/process for Cloud Backup

When Cloud Backup is used, we process and store:
• Account identifiers you provide (such as email/username) to create and access your backup account
• Authentication data (your password is encrypted; we do not store it in plaintext)
• Backup content (your ACNH Pocket Guide app data that you choose to upload, either manually or automatically based on your settings)

How we use Cloud Backup data

Cloud Backup data is used only to:
• authenticate you
• store and restore your backup content to your account
• support account recovery flows where applicable (for example, verifying identity for password reset)

To learn more about Google Firebase’s privacy practices, see:
https://firebase.google.com/support/privacy/

Retention and deletion

Data stored via Cloud Backup may be deleted by the service provider if an account is inactive for an extended period. Data may also persist for a period of time unless you request removal or delete your account (where the feature is available).
More information can be found under 'Account Deletion' of the FAQ page.

Diagnostics and Crash Reporting

We do not run third-party crash reporting that collects and transmits your crash data as a service. If you contact support, you may choose to share information (such as screenshots or logs) to help diagnose issues.

Catalog Scanner App

The Catalog Scanner application has separate behavior from ACNH Pocket Guide with respect to data processing and cloud backup services: All scanning and recognition is processed on device and no data is collected by us for the Catalog Scanner app. The catalog scanner does not provide any cloud backup services, and therefore no data is transmitted off the device.

The Catalog Scanner app does not transmit scanned images or recognized text to us. The app is fully functional without internet. When exporting to ACNH Pocket Guide, the clipboard is used and user interaction is required to transfer data.

The catalog collection is stored locally on device using Flutter Shared Preferences: https://pub.dev/packages/shared_preferences. This collection can be cleared within the application or by clearing application data and cache in device settings.

The Catalog Scanner app requires the camera solely for scanning catalog entries. Google ML Kit is used for text recognition: https://developers.google.com/ml-kit. ML Kit terms: https://developers.google.com/ml-kit/terms.
